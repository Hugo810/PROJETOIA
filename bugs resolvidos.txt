BUG 6: X-User-Id vazio causava 400 ao listar usuarios (e em outras chamadas)
======================================================================
Status: RESOLVIDO
Arquivos alterados:
  - frontend/.../services/usuario.service.ts
  - frontend/.../services/task.service.ts
  - frontend/.../usuario-list/usuario-list.ts

O que foi feito:
  - getHeaders() mudou de HttpHeaders para objeto literal
    Record<string, string> ({ 'X-User-Id': user.id.toString() }
    ou {} se usuario for null)
  - HttpClient do Angular 21 com withFetch() nao serializava
    HttpHeaders corretamente no fetch do navegador; objeto
    literal e convertido diretamente para Headers do fetch API
  - UsuarioList.ngOnInit() verifica autenticacao antes de chamar
    listar(); se nao estiver logado, redireciona para /login
  - TaskService.criarTarefa() agora inclui header X-User-Id
    (estava omitido, causando 400 ao criar tarefa)

Regra documentada:
  Todo service que depender de X-User-Id deve usar getHeaders()
  retornando Record<string, string> (objeto literal), nao
  HttpHeaders, para compatibilidade com withFetch() do Angular 21.
  Toda rota que fizer chamada autenticada deve verificar
  autenticacao antes de chamar o servico.

BUG 8: Duplicate email retornava exception Hibernate crua
======================================================================
Status: RESOLVIDO
Arquivos alterados:
  - backend/.../config/GlobalExceptionHandler.java

O que foi feito:
  - Adicionado handler para DataIntegrityViolationException que
    retorna "Email ja cadastrado." (HTTP 400) em vez da exception

Regra documentada:
